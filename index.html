Task 3

Ethical Dilemma Alternative

A major ethical issue in software development is how user data is handled. Applications often collect personal information such as names, locations, or activity data. If that information is not protected properly, it can be exposed, misused, or shared in ways users did not clearly agree to.

The most important principles in this situation are honesty, responsibility, privacy, and respect. Developers should be honest about what data is being collected, why it is needed, and how it will be used. They also have a responsibility to protect that data and avoid collecting more than what is necessary.

A better approach would be to design systems with privacy in mind from the beginning. In a project like DispatchTrack Lite, that means limiting sensitive data, protecting communication between systems, and making sure access is controlled. It also means thinking about user trust, not only technical convenience.

This alternative is better because it protects users, reduces risk, and creates a stronger professional standard. A system should not only work well. It should also be responsible and trustworthy.

Project Refinement Report

For this assignment, I selected DispatchTrack Lite as my refined project. I chose it because it gave me a good opportunity to think about both technical improvement and ethical responsibility in a realistic type of application.

One refinement was improving the structure and clarity of the user experience. I focused on making the project feel more organized and easier to understand. Another refinement was improving how I thought about frontend and backend communication, since delivery related systems depend on reliable data flow.

I also focused on realism. Instead of treating the app like a simple class exercise, I thought about how it would be used in an actual business setting. That helped me pay more attention to usability, cleaner structure, and the importance of reliability.

Ethical considerations were also part of the refinement. Because a delivery system can involve user and location related information, I reflected on privacy, controlled access, and responsible data handling. This made the project stronger because it connected technical decisions to real world responsibility.
